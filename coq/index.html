Just a Tribute to Twistor String Theory & HoTT

Channeling Jack Black: This is not the ultimate Feynman diagram model in HoTT, but a playful tribute / starting point exploring the idea.

Project Overview

This repository explores a toy model where Feynman-like diagrams are represented as right-associated binary trees. Two diagrams are considered equivalent if they have the same "leaves" — analogous to producing the same observable final particles in physics. Internals may differ, but the final outcome is what matters.

Long-term motivation: Homotopy Type Theory (HoTT) might provide a framework to represent twistor string theory Feynman diagrams as homotopy types. This project is a playful first step in that direction.

ASCII Diagram Example

Here is an example of a right-associated binary tree representing a simple diagram with three leaves:

          Node
         /    \
       Leaf1  Node
             /    \
          Leaf2   Leaf3

In this toy model, two diagrams with the same set of leaves (even if their internal structure differs) are considered equivalent.

Explore the Code

  • diagrams_equiv.v — The Coq source file modeling diagrams and equivalence with right-associated trees.
  • README.txt — Detailed explanation, CS/physics analogies, and instructions for running the Coq file.

Quick Start

  1. Install Coq.
  2. Open diagrams_equiv.v in CoqIDE or VS Code with Coq extension.
  3. Step through interactively (Coq -> Step Forward) to see examples, unit tests, and the ASCII tree printer.
  4. To compile the file from the command line, run: coqc diagrams_equiv.v. This produces a compiled .vo file in the same folder.
